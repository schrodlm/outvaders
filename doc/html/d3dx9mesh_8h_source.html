1140// This function creates atlases for meshes. There are two modes of operation,
1141// either based on the number of charts, or the maximum allowed stretch. If the
1142// maximum allowed stretch is 0, then each triangle will likely be in its own
1143// chart.
1144
1145//
1146// The parameters are as follows:
1147// pMesh - Input mesh to calculate an atlas for. This must have a position
1148// channel and at least a 2-d texture channel.
1149// uMaxChartNumber - The maximum number of charts required for the atlas.
1150// If this is 0, it will be parameterized based solely on
1151// stretch.
1152// fMaxStretch - The maximum amount of stretch, if 0, no stretching is allowed,
1153// if 1, then any amount of stretching is allowed.
1154// uWidth - The width of the texture the atlas will be used on.
1155// uHeight - The height of the texture the atlas will be used on.
1156// fGutter - The minimum distance, in texels between two charts on the atlas.
1157// this gets scaled by the width, so if fGutter is 2.5, and it is
1158// used on a 512x512 texture, then the minimum distance will be
1159// 2.5 / 512 in u-v space.
1160// dwTextureIndex - Specifies which texture coordinate to write to in the
1161// output mesh (which is cloned from the input mesh). Useful
1162// if your vertex has multiple texture coordinates.
1163// pdwAdjacency - a pointer to an array with 3 DWORDs per face, indicating
1164// which triangles are adjacent to each other.
1165// pfIMTArray - a pointer to an array with 3 FLOATs per face, describing the
1166// integrated metric tensor for that face. This lets you control
1167// the way this triangle may be stretched in the atlas. The IMT
1168// passed in will be 3 floats (a,b,c) and specify a symmetric
1169// matrix (a b) that, given a vector (s,t), specifies the
1170// (b c)
1171// distance between a vector v1 and a vector v2 = v1 + (s,t) as
1172// sqrt((s, t) * M * (s, t)^T).
1173// In other words, this lets one specify the magnitude of the
1174// stretch in an arbitrary direction in u-v space. For example
1175// if a = b = c = 1, then this scales the vector (1,1) by 2, and
1176// the vector (1,-1) by 0. Note that this is multiplying the edge
1177// length by the square of the matrix, so if you want the face to
1178// stretch to twice its
1179// size with no shearing, the IMT value should be (2, 0, 2), which
1180// is just the identity matrix times 2.
1181// Note that this assumes you have an orientation for the triangle
1182// in some 2-D space. For D3DXUVAtlas, this space is created by
1183// letting S be the direction from the first to the second
1184// vertex, and T be the cross product between the normal and S.
1185//
1186// pCallback - Since the atlas creation process can be very CPU intensive,
1187// this allows the programmer to specify a function to be called
1188// periodically, similarly to how it is done in the PRT simulation
1189// engine.
1190// fCallbackFrequency - This lets you specify how often the callback will be
1191// called. A decent default should be 0.0001f.
1192// pUserContext - a void pointer to be passed back to the callback function
1193// ppMeshOut - A pointer to a location to store a pointer for the newly created
1194// mesh.
1195// ppFacePartitioning - A pointer to a location to store a pointer for an array,
1196// one DWORD per face, giving the final partitioning
1197// created by the atlasing algorithm.
1198// ppVertexRemapArray - A pointer to a location to store a pointer for an array,
1199// one DWORD per vertex, giving the vertex it was copied
1200// from, if any vertices needed to be split.
1201// pfMaxStretchOut - A location to store the maximum stretch resulting from the
1202// atlasing algorithm.
1203// puNumChartsOut - A location to store the number of charts created, or if the
1204// maximum number of charts was too low, this gives the minimum
1205// number of charts needed to create an atlas.
1206
1207HRESULT WINAPI D3DXUVAtlasCreate(LPD3DXMESH pMesh,
