1283/* Formats
1284 * Most of these names have the following convention:
1285 * A = Alpha
1286 * R = Red
1287 * G = Green
1288 * B = Blue
1289 * X = Unused Bits
1290 * P = Palette
1291 * L = Luminance
1292 * U = dU coordinate for BumpMap
1293 * V = dV coordinate for BumpMap
1294 * S = Stencil
1295 * D = Depth (e.g. Z or W buffer)
1296 * C = Computed from other channels (typically on certain read operations)
1297 *
1298 * Further, the order of the pieces are from MSB first; hence
1299 * D3DFMT_A8L8 indicates that the high byte of this two byte
1300 * format is alpha.
1301 *
1302 * D3DFMT_D16_LOCKABLE indicates:
1303 * - An integer 16-bit value.
1304 * - An app-lockable surface.
1305 *
1306 * D3DFMT_D32F_LOCKABLE indicates:
1307 * - An IEEE 754 floating-point value.
1308 * - An app-lockable surface.
1309 *
1310 * All Depth/Stencil formats except D3DFMT_D16_LOCKABLE and D3DFMT_D32F_LOCKABLE indicate:
1311 * - no particular bit ordering per pixel, and
1312 * - are not app lockable, and
1313 * - the driver is allowed to consume more than the indicated
1314 * number of bits per Depth channel (but not Stencil channel).
1315 */

1318 ((DWORD)(BYTE)(ch0) | ((DWORD)(BYTE)(ch1) << 8) | \
